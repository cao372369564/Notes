=============================================== 一、vue 挂载元素方式 1. new Vue({ el : '#app' }) 2. vm.$mount('#app') ======================================================= 二、template 会替换掉挂载的元素，页面呈现的内容会是template 对应的value值变成真正的dom. template:String -> AST({}) -> render -> VNode -> 真实数据 1. 首先 template字符串会被解析成为抽象语法数(AST),然后传 入渲染函数render,然后创造一个虚拟节点VNode,最后才拿到真实数据。 ======================================================= 三、vue渲染el挂载元素 1. 首先会初始化，检查有没有el元素,有的话就检查template模板， 若template模板也有,就会自动执行'第二条'. 若el不存在,则会检查有没有调用 $mount方法，若有也会再检查 看template模板，然后执行'第二条',若$mount也没有,这个vue实例 就不会在进行任何操作了,页面也就渲染不出来。 2. 拿到el元素之后,检查实例中有没有template模板,若没有， 就会找这个挂载元素的outerHTML,这个outerHTML也表示这个el元素 节点,但它是字符串格式的,并不是真正的dom.然后vue就会创建template, 并把el元素的outerHTML放在template上 =================================================================== 四、render函数,这个render函数接受一个参数,这个参数可以得到一个VNode render (createElement){ return createElement('div',{ style : { color : 'red', fontSize '12px' }, class : { red : this.red, green : this.green } },[ createElement('h1'), createElement('h2'), createElement('h3') ]) } 优先级 ： render > template > el ====================================================================== =================================vue生命周期=========================== new一个vue()，就开始了生命周期的历程, 1. 初始化事件和生命周期 2. beforeCreate事件执行时,data,methods等对象还不存在 3. 初始化注入，意思是这一步进行了data,methods等对象的创建并初始化 4. created事件执行，vue实例已经被正式被创建完了,用created就可以执行到data, methods内部的值了 5. 判断有没有el属性 6. 判断有没有template 7. beforeMount事件执行 8. 创建template中的元素并替换掉el挂载的元素 9. mounted 事件执行， 当数据改变,先执行beforeUpdate事件,虚拟dom渲染完成 再执行Updated事件,updated执行完,映射到页面数据 10. beforeDestroy事件，销毁前执行的方法 11. destroyed事件，销毁vue实例之后执行 此时vue实例挂载彻底失效
